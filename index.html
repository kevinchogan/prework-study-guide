HTML

  • The DOCTYPE declaration tells the browser what document type it should expect when it is opened.
  • The html tag represents the root, or start, of the HTML document.
    • The lang attribute tells the browser the language for the webpage (en for English).
  • The head element contains information about the webpage.
    • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
      • The charset element sets our character encoding.
      • The http-equiv element sets an HTTP header for content.
      • The viewport element sets the default view for our webpage.
    • The title element defines the title for a webpage. This element is required and is extremely important for Search Engine Optimization (SEO)
    • The style element defines the CSS (aka Cascading Style Sheets) styles associated with a website.
    • The base element defines the base URL for a webpage.
    • The link element connects an external resource to the HTML document.
  • The body element represents the visible content shown to the user.
    • The section elements are semantic elements that define a section in a document that contains thematic content.
    • The h1, h2, h3, h4, h5, and h6 elements represent the level of heading a given text block represents.
    • The ul tag creates an unordered list of items.
    • The ol tag creates an ordered list of items.
    • The li tag creates a list items.
    • The img element contains information about images that are displayed on the webpage.
      • The src attribute defines the location of the image.
      • The alt attribute defines the alternate text for the image
      • The width attribute defines the display width of the image.
      • The height attribute defines the display height of the image.
    • The p element represents a paragraph or block of text.
    • The br element creates a line of empty space, or a line break, between two blocks of content.
    • the a element (which stands for anchor) creates a link to a webpage.
      • The href attribute points to the URL for the link
  • The script element embeds client-side scripts in the HTML document.

Git

  • CLI
    • The ls command shows the list of files and directories on our computer.
    • The cd command allows us to change directories.
    • The pwd command, which is short for "print working directory," will print out the directory that we are currently in.
    • The mkdir command will create a directory.
    • The touch command will create a new file.
    • Type code . to open VS Code in the current directory.
    • Git Commands (Ensure git precedes all following commands)
      • The checkout command is to move the working branch to a new branch (-b to start a new branch).
      • Use the status command to check what branch you are currently on.
      • The add command adds modifications in the current working branch to the staging area (use the -A flag to add all changes).
      • The commit command commits changes in the staging area (use -m to add a message that describes what is in the commit).
      • Use the pull command to receive a branch's modifications into the local environment.
        • Include the source (origin will pull from the Github repo) and the branch (either main or a created branch).
      • Use push to push the changes you have locally to the remote GitHub branch.
